# 版本分享说明目录设计

## 目标

新增一个文档目录，用于存放版本分享说明。

## 设计

创建 `docs/release-shares/` 作为顶层文档分类。每个版本使用一个独立目录，例如 `docs/release-shares/v0.1/`。只有当用户明确要求为某个版本生成、同步或创建分享说明时，才在对应版本目录下新增或更新 Markdown 文件。

该目录刻意与 `docs/superpowers/`、发布计划、验收文档和路线图分开，避免将面向分享的叙述材料与实现计划、项目跟踪文档混在一起。

同一个版本可以包含多篇分享说明。例如 v0.1 可以同时包含总览、Agent Loop 机制、测试说明等不同主题文档。

## 手动创建规则

分享说明不会自动生成，也不会从其他文档自动同步。只有在用户指定目标版本并明确要求创建、同步或更新分享说明时，才新增或修改对应文档。

内容需要由用户审阅和确认后，才视为最终版本。

## 目录和文件命名

使用版本号作为目录名，并在目录内按主题命名文件：

```text
docs/release-shares/v0.1/
  README.md
  overview.md
  agent-loop.md

docs/release-shares/v0.2/
  README.md
  overview.md
```

每个版本目录的 `README.md` 作为索引，列出该版本下所有分享文档及其来源。主题文件名使用简短英文 slug，例如 `overview.md`、`agent-loop.md`。

## 非目标

- 不做自动同步。
- 不新增发布说明生成脚本。
- 不修改构建或 CLI 行为。
