# v0.1 验收清单

ForgeCode v0.1 的目标是证明最小本地 agent loop 已经成立：它能接收任务、进入可追踪执行循环、调用工具、运行验证，并通过 micro-benchmark 做基础复现。

## 功能验收

v0.1 完成时必须满足：

- `forgecode run <task>` 能运行本地 agent loop。
- agent session 会记录结构化 trace。
- trace 至少能记录 `plan`、`tool_call`、`tool_result` 和 `final`。
- loop 能通过 workspace tools 检查文件。
- loop 能在 workspace 内写入范围受限的文件修改。
- workspace path policy 会拒绝 workspace root 外的路径。
- loop 能运行验证命令。
- 失败验证输出会被记录进 trace。
- deterministic/scripted provider 能驱动可复现测试。
- micro-benchmark harness 至少能运行 3 个 smoke task。

## 验证命令

v0.1 完成时以下命令必须通过：

```bash
npm test
npm run typecheck
npm run build
npm run bench:micro
```

## 当前 Smoke Tasks

`benchmarks/micro/tasks.json` 当前包含：

- `readme-smoke`：运行 `npm test`。
- `typecheck-smoke`：运行 `npm run typecheck`。
- `build-smoke`：运行 `npm run build`。

## 不属于 v0.1 的内容

以下能力不作为 v0.1 完成条件：

- 真实 LLM provider。
- Claude Code-style 的完整终端交互体验。
- 用户已有修改保护。
- destructive command 权限确认。
- 上下文选择和压缩策略。
- 外部 benchmark 子集。

这些能力会进入 v0.2 和 v0.3。
